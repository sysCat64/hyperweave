---
title: "Install Session Telemetry"
description: "Set up HyperWeave to automatically generate visual session receipts for Claude Code, Codex, and Antigravity conversations."
---

HyperWeave can parse Claude Code, Codex, and Antigravity session transcripts and render structured telemetry artifacts -- receipts and rhythm strips. This tutorial sets up automatic receipt generation via each agent runtime's hook system.

## Prerequisites

- HyperWeave installed
- At least one supported agent runtime installed and configured: Claude Code, Codex, or Antigravity
- At least one completed session transcript:
  - Claude Code: `~/.claude/projects/.../*.jsonl`
  - Codex: `~/.codex/sessions/.../*.jsonl`
  - Antigravity: `~/.gemini/antigravity/brain/.../*.jsonl`

## Step 1: Install the Hook

HyperWeave ships with a one-command hook installer. Without `--runtime`, it auto-detects installed runtimes and registers each one it finds:

```bash
hyperweave install-hook
```

You can also install for one runtime explicitly:

```bash
hyperweave install-hook --runtime claude-code
hyperweave install-hook --runtime codex
hyperweave install-hook --runtime antigravity
```

This installs:

| Runtime | Hook location | Event |
|---|---|---|
| Claude Code | `~/.claude/settings.json` | `SessionEnd` |
| Codex | `~/.codex/hooks.json` plus `[features] hooks = true` in `~/.codex/config.toml` | `Stop` |
| Antigravity | `~/.gemini/config/hooks.json` | `Stop` |

<Info>
  Hooks run `hyperweave session receipt`. The command reads the transcript path from the runtime's stdin JSON and writes a receipt SVG to `.hyperweave/receipts/`. Antigravity hooks use the first `workspacePaths` entry so receipts land in the project workspace instead of the global config directory.
</Info>

## Step 2: Verify the Hook

Run the read-only doctor command:

```bash
hyperweave doctor
```

You should see each detected runtime and its hook status, for example:

```text
hyperweave doctor — v0.x.y

Runtimes:
  ✓ claude-code: hook registered -- hyperweave session receipt
  ✓ codex: hook registered -- hyperweave session receipt
  ✓ antigravity: hook registered -- hyperweave session receipt
```

If you inspect config files directly, Claude Code should contain a `SessionEnd` hook like this:

```json
{
  "hooks": {
    "SessionEnd": [
      {
        "hooks": [
          {
            "type": "command",
            "command": "hyperweave session receipt",
            "timeout": 10
          }
        ]
      }
    ]
  }
}
```

## Step 3: Run a Test Session

Start a supported agent session, ask a question, then end the conversation. After the hook runs, check for the receipt in the workspace:

```bash
ls .hyperweave/receipts/
```

You should see an SVG file named from the session timestamp and session title or prompt.

## Step 4: Manual Receipt Generation

You can also generate receipts manually from any supported transcript. HyperWeave auto-detects the runtime from the JSONL shape:

```bash
hyperweave session receipt ~/.claude/projects/<project>/sessions/<session-id>.jsonl
hyperweave session receipt ~/.codex/sessions/.../<session-id>.jsonl
hyperweave session receipt ~/.gemini/antigravity/brain/.../<session-id>.jsonl
```

Or parse the raw telemetry data:

```bash
hyperweave session parse path/to/session.jsonl
```

This outputs the telemetry contract as JSON -- useful for debugging or custom visualization.

## Step 5: Render a Rhythm Strip

Rhythm strips show tool call density over time:

```bash
hyperweave session strip path/to/session.jsonl -o strip.svg
```

## Step 6: Backfill Antigravity Receipts

Antigravity sessions can leave existing transcripts under `~/.gemini/antigravity/brain/`. Backfill missing or stale receipts into the current workspace:

```bash
hyperweave session sync --runtime antigravity
```

Use `--force` to re-render receipts even when an existing SVG is newer than the transcript:

```bash
hyperweave session sync --runtime antigravity --force
```

## What the Telemetry Pipeline Does

The telemetry system processes session transcripts in 5 passes:

1. **Tool calls**: Extracts tool names, arguments, durations, outcomes
2. **Outcomes**: Classifies success, failure, and partial results
3. **User text**: Captures prompts, corrections, and feedback signals
4. **Agent spans**: Identifies reasoning blocks and decision points
5. **Turn durations**: Computes latency and throughput metrics

Then applies:
- **3-signal stage detector**: Temporal (0.3), class shift (0.4), explicit (0.3) with 0.6 threshold
- **Dual-signal correction classifier**: Lexical patterns + behavioral signals
- **Cost computation**: Token counts and pricing

All configuration lives in YAML files under `data/telemetry/`:
- `tool-colors.yaml` -- tool name to color mapping
- `runtimes/*.yaml` -- runtime detection rules, parser module, provider identity, and tool class mapping
- `stage-labels.yaml` -- stage ID to label mapping
- `stage-config.yaml` -- detector weights and thresholds

## What You Learned

- `hyperweave install-hook` sets up automatic session receipts for detected supported runtimes
- Receipts are rendered to `.hyperweave/receipts/` after each session
- Manual generation works with `hyperweave session receipt <path>`
- `hyperweave session sync --runtime antigravity` backfills Antigravity receipts
- The telemetry pipeline uses a 5-pass parser with 3-signal stage detection
