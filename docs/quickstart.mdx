## Install

<CodeGroup>
```bash uv (CLI + HTTP + MCP)
uv add "hyperweave[all]"
```

```bash pip (CLI + HTTP + MCP)
pip install "hyperweave[all]"
```

```bash uv (CLI only)
uv add hyperweave
```
</CodeGroup>

Requires Python 3.12+.

Install `hyperweave` for local CLI composition. Use `hyperweave[all]` when you
also want the HTTP server and MCP server available from the same environment.

## Compose a Badge

<Tabs>
  <Tab title="CLI">
    ```bash
    hyperweave compose badge "build" "passing" -g brutalist -o badge.svg
    ```
    Open `badge.svg` in a browser, or omit `-o` to print the SVG to stdout:
    ```bash
    hyperweave compose badge "build" "passing" -g brutalist
    ```
  </Tab>
  <Tab title="Python">
    ```python
    from hyperweave.compose.engine import compose
    from hyperweave.core.models import ComposeSpec

    spec = ComposeSpec(
        type="badge",
        genome_id="brutalist",
        title="build",
        value="passing",
        state="passing",
    )
    result = compose(spec)
    print(result.svg)  # Self-contained SVG string
    ```
  </Tab>
  <Tab title="HTTP">
    ```bash
    # Start the server
    hyperweave serve --port 8000

    # URL grammar
    curl http://localhost:8000/v1/badge/build/passing/brutalist.static

    # POST
    curl -X POST http://localhost:8000/v1/compose \
      -H "Content-Type: application/json" \
      -d '{"type":"badge","genome":"brutalist","title":"build","value":"passing"}'
    ```
  </Tab>
  <Tab title="MCP">
    Add to your MCP configuration:
    ```json
    {
      "mcpServers": {
        "hyperweave": {
          "command": "hyperweave",
          "args": ["mcp"]
        }
      }
    }
    ```
    Then call the tool:
    ```
    hw_compose(type="badge", title="build", value="passing", genome="brutalist")
    ```
  </Tab>
</Tabs>

## Add Motion

Apply a border animation to badges and strips:

```bash
hyperweave compose badge "build" "passing" -g brutalist -m chromatic-pulse --regime permissive
```

Under the default `normal` regime, non-CIM-safe motions fall back to `static`.
Use `--regime permissive` when you intentionally want the animated border
specimen.

## Switch Genomes

Core frames render across the built-in genomes. Same content, different
aesthetic:

```bash
# Brutalist: dark field, emerald signal, sharp corners
hyperweave compose badge "build" "passing" -g brutalist

# Chrome: deep void, metallic blue, smooth curves
hyperweave compose badge "build" "passing" -g chrome

# Automata and Primer are available too
hyperweave compose badge "build" "passing" -g automata
hyperweave compose badge "build" "passing" -g primer
```

## Compose an Artifact Kit

Generate an entire artifact set in one call:

```bash
hyperweave kit readme \
  -g brutalist \
  --project "my-project" \
  --badges "build:passing,version:v0.6.3,coverage:95%" \
  --social "github,discord,x" \
  --output ./assets/
```

This writes badge-build, badge-version, badge-coverage, strip, divider,
icon-github, icon-discord, and icon-x SVG files to `./assets/`.

MCP clients can request the same kit as in-memory SVG strings:

```
hw_kit(
  type="readme",
  genome="brutalist",
  project="my-project",
  badges="build:passing,version:v0.6.3,coverage:95%",
  social="github,discord,x"
)
```

## Data-Driven Artifacts

Use the shared `--data` token grammar for live or structured values:

```bash
hyperweave compose badge "version" --data kv:VERSION=0.6.3 -g brutalist
hyperweave compose strip "project" --data kv:STARS=42K,kv:VERSION=0.6.3 -g chrome
hyperweave compose marquee-horizontal --data text:NEW,kv:STARS=42K,text:DOWNLOADS -g primer
```

Swap `kv:` tokens for live tokens such as `gh:openai/codex.stars` or
`pypi:fastmcp.version` when you want provider-backed values. Supported
providers include GitHub, PyPI, npm, arXiv, HuggingFace, Docker Hub, crates.io,
OpenSSF Scorecard, and DORA metrics.
