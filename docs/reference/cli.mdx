The HyperWeave CLI is built with Typer. All commands output SVG to stdout by default (pipe-friendly) or to a file with `-o`.

```bash
hyperweave --help
```

## Commands

### `compose`

Compose a single artifact.

```bash
hyperweave compose <frame_type> [title] [value] [options]
```

| Argument | Description |
|---|---|
| `frame_type` | Frame: `badge`, `strip`, `icon`, `divider`, `marquee-horizontal`, `stats`, `chart`, `matrix` |
| `title` | Primary text (label, identity, username, owner/repo, etc.) |
| `value` | Secondary text or chart subtype (for example, `stars`) |

| Option | Short | Default | Description |
|---|---|---|---|
| `--genome` | `-g` | `brutalist` | Genome slug |
| `--genome-file` | | `None` | Path to a local genome JSON file (validated, bypasses built-in registry) |
| `--state` | `-s` | `active` | Semantic status |
| `--motion` | `-m` | `static` | Motion primitive |
| `--glyph` | | `""` | Glyph ID |
| `--glyph-mode` | | `auto` | Glyph rendering mode |
| `--regime` | | `normal` | Policy regime |
| `--size` | | `default` | Frame size |
| `--variant` | | `""` | Variant slug from the genome whitelist |
| `--pair` | | `""` | Cellular paradigm pairing modifier for bifamily frames |
| `--state-glyph-shape` | | `""` | Badge state-indicator shape: `square`, `circle`, or `diamond` |
| `--shape` | | `""` | Icon frame shape: `square`, `circle` |
| `--divider-variant` | | `zeropoint` | Divider visual variant |
| `--direction` | | `ltr` | Marquee scroll direction |
| `--data` | | `""` | Data tokens for marquee and live-data composition |
| `--spec-file` | | `None` | MatrixSpec JSON file for the matrix frame |
| `--preset` | | `""` | Server-known matrix preset, such as `connectors` |
| `--markdown-out` | | `None` | Also write the matrix frame's GFM markdown shadow |
| `--glyph-tint` | | `""` | Matrix glyph fill selection: `ink`, `brand`, or `full` |
| `--metrics` | | `""` | Strip metrics (overrides value) |
| `--output` | `-o` | stdout | Output file path |

**Examples:**

```bash
hyperweave compose badge "build" "passing" -g brutalist -s passing
hyperweave compose strip "readme-ai" "" -g brutalist --metrics "STARS:2.9k,FORKS:278"
hyperweave compose icon "" "" --glyph github -g brutalist
hyperweave compose divider "" "" --divider-variant void -g brutalist

# live-data and structured-data frames
hyperweave compose stats eli64s -g chrome -o stats.svg
hyperweave compose chart stars eli64s/readme-ai -g brutalist -o chart.svg
hyperweave compose matrix --preset connectors -g primer --markdown-out table.md

# Custom genome from local JSON
hyperweave compose badge "DEPLOY" "live" --genome-file ./my-genome.json
```

### `kit`

Compose a full artifact kit.

```bash
hyperweave kit [kit_type] [options]
```

| Option | Short | Default | Description |
|---|---|---|---|
| `kit_type` | | `readme` | Kit type |
| `--genome` | `-g` | `brutalist` | Genome slug |
| `--project` | | `""` | Project name, used as the strip identity |
| `--badges` | | `""` | Badge pairs: `"build:passing,version:v0.6.3"` |
| `--social` | | `""` | Social icons: `"github,discord,x"` |
| `--output` | `-o` | `.` | Output directory |

### `render`

Render a telemetry artifact from a data contract.

```bash
hyperweave render --template <name> --data <path> [-o output]
```

| Option | Description |
|---|---|
| `--template` | Template name: `receipt`, `rhythm-strip` |
| `--data` | Path to data contract JSON file |
| `--output` / `-o` | Output file path (stdout if omitted) |

### `session`

Session telemetry: parse transcripts, render receipts and rhythm strips, or sync receipt files from a runtime transcript directory.

```bash
hyperweave session <action> [transcript_path] [-o output]
```

| Action | Description |
|---|---|
| `receipt` | Parse transcript and render a receipt card |
| `strip` | Parse transcript and render a rhythm strip |
| `parse` | Parse transcript and output JSON to stdout |
| `sync` | Backfill missing or stale receipts from a runtime transcript directory |

| Option | Default | Description |
|---|---|---|
| `--output` / `-o` | `None` | Output file for `receipt` or `strip`; receipts default to `.hyperweave/receipts/` when omitted |
| `--output-dir` | `.hyperweave/receipts` | Receipt directory for `session sync` |
| `--genome` | auto | Pin telemetry skin (`cream`, `voltage`, `claude-code`, `codex`, or full `telemetry-*` slug) |
| `--runtime` | `antigravity` | Runtime for `session sync`; currently supports `antigravity` |
| `--force` | `false` | Re-render synced receipts even when the existing SVG is newer |

HyperWeave auto-detects Claude Code, Codex, and Antigravity transcripts from their JSONL shape. When invoked as a hook, `session receipt` reads `transcript_path` or `transcriptPath` from stdin JSON; Antigravity hook payloads also use `workspacePaths` so receipts land in the project workspace.

Examples:

```bash
hyperweave session receipt path/to/session.jsonl
hyperweave session strip path/to/session.jsonl -o rhythm.svg
hyperweave session parse path/to/session.jsonl
hyperweave session sync --runtime antigravity
hyperweave session sync --runtime antigravity --force --output-dir .hyperweave/receipts
```

### `live`

Compose a badge with live data from a provider.

```bash
hyperweave live <provider> <identifier> <metric> [options]
```

| Argument | Description |
|---|---|
| `provider` | `github`, `pypi`, `npm`, `arxiv`, `huggingface`, `docker`, `crates`, `scorecard`, `dora` |
| `identifier` | Resource ID (`owner/repo`, `package-name`, `paper-id`, container image, or crate name) |
| `metric` | Provider metric such as `stars`, `forks`, `version`, `downloads`, `pull_count`, `score`, or `deploy_frequency` |

### `genomes`

List or inspect genomes.

```bash
hyperweave genomes [genome_id] [--ids-only]
```

### `serve`

Start the HTTP server.

```bash
hyperweave serve [--port 8000] [--host 0.0.0.0] [--reload]
```

### `install-hook`

Install session-receipt hooks for supported agent runtimes.

```bash
hyperweave install-hook
```

Without `--runtime`, HyperWeave auto-detects initialized or installed agent runtimes and registers each one it finds. Pass `--runtime <name>` to scope installation, or `--runtime all` to force every supported runtime.

| Option | Default | Description |
|---|---|---|
| `--genome` | auto | Pin telemetry skin for every generated receipt |
| `--runtime` | auto | Runtime to install: `claude-code`, `codex`, `antigravity`, or `all` |

| Runtime | Hook location | Event |
|---|---|---|
| Claude Code | `~/.claude/settings.json` | `SessionEnd` |
| Codex | `~/.codex/hooks.json` plus `[features] hooks = true` in `~/.codex/config.toml` | `Stop` |
| Antigravity | `~/.gemini/config/hooks.json` | `Stop` |

Examples:

```bash
hyperweave install-hook
hyperweave install-hook --runtime antigravity
hyperweave install-hook --runtime all --genome voltage
```

### `doctor`

Diagnose telemetry hook wiring, transcript discovery, and recent receipt output across supported agent runtimes. This command is read-only and exits zero even when it reports warnings.

```bash
hyperweave doctor
```

It reports:

| Section | Description |
|---|---|
| Runtimes | Detection state and hook registration for Claude Code, Codex, and Antigravity |
| Transcripts | JSONL transcript counts and most recent transcript timestamp per runtime |
| Receipts | Recent and total `.hyperweave/receipts/*.svg` files in the current directory |

### `validate-genome`

Validate a genome JSON file against its profile contract schema. Checks required `--dna-*` fields, type constraints, and WCAG AA contrast ratios.

```bash
hyperweave validate-genome <genome_path> [--profile <profile_id>]
```

| Option | Description |
|---|---|
| `genome_path` | Path to the genome JSON file to validate |
| `--profile` | Profile to validate against (defaults to `genome.profile`) |

### `mcp`

Start the HyperWeave MCP server. Used when configuring the MCP client via the shipped binary.

```bash
hyperweave mcp [--transport stdio]
```

| Option | Default | Description |
|---|---|---|
| `--transport` | `stdio` | Transport: `stdio`, `http`, `sse`, `streamable-http` |

### `version`

Print the HyperWeave version.

```bash
hyperweave version
```
