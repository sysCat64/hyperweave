A genome is a portable, machine-readable aesthetic specification. It defines
color, material, typography, rhythm, compatible motions, and optional variants.
Every artifact inherits its visual identity from a genome.

Built-in genome JSON files live under `src/hyperweave/data/genomes/`.

## Motion Compatibility

Genome compatibility is checked before frame compatibility. A motion must pass
both checks before it can render visibly; CIM regime behavior can still correct
non-CIM motions to `static`.

| Genome | Compatible Motions |
|---|---|
| `brutalist` | `static`, `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement`, `rimrun` |
| `chrome` | `static`, `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement`, `rimrun` |
| `automata` | `static`, `chromatic-pulse`, `corner-trace`, `rimrun` |
| `primer` | `static`, `chromatic-pulse`, `corner-trace` |
| `telemetry-*` | `static` |

`rimrun` is still badge-scoped by frame compatibility, even when a genome lists
it as compatible.

## Genome Specification Fields

Every genome config contains a common core of fields. Individual genomes can add
variant, material, glyph, telemetry, or frame-specific fields.

| Field | Type | Description |
|---|---|---|
| `id` | `str` | Unique genome slug |
| `name` | `str` | Display name |
| `category` | `"dark" \| "light"` | Base theme category |
| `profile` | `str` | Structural profile reference |
| `surface_0` | color | Primary surface |
| `surface_1` | color | Alternate surface |
| `surface_2` | color | Deep surface |
| `ink` | color | Primary text color |
| `ink_secondary` | color | Muted text color |
| `ink_on_accent` | color | Text color on accent backgrounds |
| `accent` | color | Primary signal color |
| `accent_complement` | color | Secondary signal color |
| `stroke` | color | Border or separator color |
| `corner` | `str` | Corner radius token |
| `rhythm_base` | `str` | Base animation timing |
| `density` | `str` | Visual density factor |
| `compatible_motions` | `list[str]` | Motion allowlist; must include `static` |

## CSS Variable Mapping

The compose assembler maps genome fields into CSS custom properties before SVG
templates render.

| Genome Field | CSS Variable |
|---|---|
| `surface_0` | `--dna-surface` |
| `surface_1` | `--dna-surface-alt` |
| `surface_2` | `--dna-surface-deep` |
| `ink` | `--dna-ink-primary` |
| `ink_secondary` | `--dna-ink-muted` |
| `ink_on_accent` | `--dna-ink-on-accent` |
| `accent` | `--dna-signal` |
| `accent_complement` | `--dna-signal-dim` |
| `stroke` | `--dna-border` |
| `corner` | `--dna-corner` |
| `rhythm_base` | `--dna-rhythm-base` |

Templates consume `var(--dna-*)` values instead of hardcoding colors.

## Variants

Several artifact genomes ship variants. The `variants` list is the allowed slug
set. Depending on the genome, a variant can resolve through sparse
`variant_overrides` entries or through tone primitives, while retaining the same
profile and base motion allowlist.

Use `--variant <slug>` on the CLI or `variant=<slug>` in URL grammar routes.

```bash
hyperweave compose badge "build" "passing" -g primer --variant porcelain
curl "localhost:8000/v1/badge/build/passing/chrome.static?variant=horizon"
```

## Rhythm System

Genomes expose rhythm tokens to animation templates.

| Rhythm Field | CSS Variable | Notes |
|---|---|---|
| `rhythm_base` | `--dna-rhythm-base` | Primary timing token |
| `rhythm_slow` | `--dna-rhythm-slow` | Optional slower timing token |
| `rhythm_fast` | `--dna-rhythm-fast` | Optional faster timing token |
