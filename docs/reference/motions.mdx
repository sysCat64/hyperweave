title: "Motions Reference"

Motion primitives add governed animation metadata and SVG overlays to artifacts. The current public vocabulary is the `MotionId` enum: `static` plus five SMIL border motions.

## Motion Categories

### Static

`static` is the default path. It emits no motion CSS, no motion class, and short-circuits motion validation.

| ID | CIM | Applies To | Behavior |
|---|---|---|---|
| `static` | Yes | All frames | No animation; no border overlay |

### Border Motions (SMIL)

Border motions are SVG-native SMIL fragments rendered from `src/hyperweave/templates/motions/border/`. They can attach to badge, strip, and icon frames when both genome compatibility and frame compatibility allow them.

| ID | CIM | Applies To | Description |
|---|---|---|---|
| `chromatic-pulse` | No | `badge`, `strip`, `icon` | Chromatic gradient cycling with stroke-width and opacity modulation |
| `corner-trace` | No | `badge`, `strip`, `icon` | Single runner tracing the rounded frame perimeter |
| `dual-orbit` | No | `badge`, `strip`, `icon` | Two perimeter runners offset by half a circuit |
| `entanglement` | No | `badge`, `strip`, `icon` | Coupled chromatic gradient and paired dash traversal |
| `rimrun` | No | `badge` | Badge-sized frosted-prism rim runners with refraction and glare layers |

`rimrun` is registry-scoped to `badge` under normal compatibility checks. The template can receive seam geometry, but the motion registry is the source of truth for default frame eligibility.

## Frame Compatibility

Motion compatibility is checked after genome compatibility. `static` is always valid.

| Frame | Allowed Motions Under `normal` |
|---|---|
| `badge` | `static`, `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement`, `rimrun` |
| `strip` | `static`, `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement` |
| `icon` | `static`, `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement` |
| `divider` | `static` |
| `marquee-horizontal` | `static` |
| `receipt` | `static` |
| `rhythm-strip` | `static` |
| `stats` | `static` |
| `chart` | `static` |
| `matrix` | `static` |

## CIM Compliance

The Compositional Integrity Model (CIM) defines which motions are safe for use in governed contexts.

**CIM-compliant motions**:
`static`

**Non-CIM motions**:
`chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement`, `rimrun`

### Regime Behavior

| Regime | Policy-Lane Behavior | Frame Compatibility Behavior |
|---|---|---|
| `normal` | Non-CIM motions are corrected to `static` | Incompatible frame requests resolve to `static` |
| `permissive` | Non-CIM motions are allowed with logged violations | Incompatible frame requests are allowed with a reason string |
| `ungoverned` | Policy-lane checks are skipped | Unknown or incompatible motions can pass validation, but visible output still requires a registered config/template |

## Template Structure

Motion templates live under `src/hyperweave/templates/motions/`:

```
src/hyperweave/templates/motions/
  border/
    chromatic-pulse.svg.j2
    corner-trace.svg.j2
    dual-orbit.svg.j2
    entanglement.svg.j2
    rimrun.svg.j2
```

The runtime scans the root motion registry and the `border/` registry subdirectory. Border templates produce SMIL `<animate>` elements and expose `defs` plus `overlay` fragments to the frame template.

## Motion YAML Config

Each motion has a YAML config under `src/hyperweave/data/motions/`:

```yaml
# src/hyperweave/data/motions/border/chromatic-pulse.yaml
id: chromatic-pulse
name: Chromatic Pulse
category: smil
cim_compliant: false
animated_properties:
  - stop-color
  - stroke-width
  - opacity
applies_to: [badge, strip, icon]
```

The `static.yaml` config lives at the root of `src/hyperweave/data/motions/`; border motion configs live in `src/hyperweave/data/motions/border/`.

## Seam Handling

Border overlays receive computed frame geometry from Python before the Jinja template renders.

| Context Value | Used By | Purpose |
|---|---|---|
| `w`, `h`, `rx` | All border motions | Size and corner radius of the traced frame |
| `perim` | `corner-trace`, `dual-orbit`, `entanglement` | Rounded-rectangle perimeter used for dash travel |
| `lp_w`, `right_x` | `rimrun` | Badge panel boundaries for left/right rim paths |
| `seam_positions` | `rimrun` | Optional strip divider coordinates for seam-aware path generation |
