---
title: "MCP Tools Reference"
description: "Reference for HyperWeave's 4 MCP tools and 3 resources via FastMCP v3."
---

HyperWeave exposes a FastMCP v3 server with 4 tools and 3 resources. Configure it in your MCP client:

```json
{
  "mcpServers": {
    "hyperweave": {
      "command": "hyperweave",
      "args": ["mcp"]
    }
  }
}
```

## Tools

### `hw_compose`

Compose any HyperWeave artifact. Returns self-contained SVG string.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `type` | `str` | `"badge"` | Frame type |
| `title` | `str` | `""` | Primary text |
| `value` | `str` | `""` | Secondary text |
| `genome` | `str` | `"brutalist"` | Genome slug |
| `state` | `str` | `"active"` | Semantic status |
| `motion` | `str` | `"static"` | Motion primitive |
| `glyph` | `str` | `""` | Glyph ID |
| `glyph_mode` | `str` | `"auto"` | Glyph rendering mode |
| `regime` | `str` | `"normal"` | Policy regime |
| `size` | `str` | `"default"` | Frame size |
| `shape` | `str` | `""` | Icon frame shape: `square`, `circle` |
| `variant` | `str` | `""` | Variant slug from the genome whitelist |
| `pair` | `str` | `""` | Cellular pairing modifier for bifamily frames |
| `state_glyph_shape` | `str` | `""` | Badge state-indicator shape override |
| `divider_variant` | `str` | `"zeropoint"` | Divider variant |
| `direction` | `str` | `"ltr"` | Marquee direction |
| `speeds` | `list[float] \| None` | `None` | Marquee scroll speed multipliers |
| `data` | `str` | `""` | Unified data-token grammar for live values |
| `telemetry_data` | `dict \| None` | `None` | Telemetry contract |
| `genome_override` | `dict \| None` | `None` | Inline genome dict (equivalent to CLI `--genome-file`) |
| `connector_data` | `dict \| None` | `None` | Pre-fetched connector payload for stats/chart, or matrix adapter payload |
| `stats_username` | `str` | `""` | GitHub username for `stats` frame |
| `chart_owner` | `str` | `""` | GitHub owner for `chart` frame |
| `chart_repo` | `str` | `""` | GitHub repo for `chart` frame |
| `matrix` | `dict \| None` | `None` | Matrix frame table IR |
| `glyph_tint` | `str` | `""` | Matrix glyph fill selection |
| `render_target` | `str` | `"svg"` | `svg`, or `markdown` for matrix frames |

**Content by frame type:**

| Frame | `title` | `value` | Notes |
|---|---|---|---|
| badge | Left panel text | Right panel text | |
| strip | Identity text | Metrics: `"STARS:2.9k,FORKS:278"` | |
| icon | (unused) | (unused, use `glyph`) | |
| divider | (unused) | (unused, use `divider_variant`) | |
| marquee-horizontal | Pipe-separated text | (unused) | Can also use `data` tokens |
| receipt | (unused) | (unused, use `telemetry_data`) | |
| rhythm-strip | (unused) | (unused, use `telemetry_data`) | |
| stats | (unused) | (unused) | Use `stats_username` + `connector_data` |
| chart | (unused) | (unused) | Use `chart_owner`/`chart_repo` + `connector_data` |
| matrix | (unused) | (unused) | Use `matrix`, `connector_data={"matrix_adapter": "connector-registry"}`, or `data` tokens |

<Note>
`hw_compose` can resolve live values inline when `data` tokens are provided. For `stats`/`chart` frames, pass `connector_data` when the caller already has a pre-fetched payload. Matrix callers can pass a `matrix` table directly, or use `connector_data={"matrix_adapter": "connector-registry"}` for the generated connector matrix.
</Note>

Examples:

```
hw_compose(type="badge", title="build", value="passing", genome="brutalist")
```

```python
await hw_compose(
    type="matrix",
    genome="primer",
    connector_data={"matrix_adapter": "connector-registry"},
    render_target="markdown",
)
```

The first call returns a self-contained SVG badge. The matrix call returns the
GFM markdown projection instead of SVG.

### `hw_live`

Compose a badge with live data fetched from a provider.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `provider` | `str` | **required** | `github`, `pypi`, `npm`, `arxiv`, `huggingface`, `docker`, `crates`, `scorecard`, `dora` |
| `identifier` | `str` | **required** | Resource ID |
| `metric` | `str` | **required** | Metric name |
| `genome` | `str` | `"brutalist"` | Genome slug |
| `glyph` | `str` | `""` | Glyph ID |
| `state` | `str` | `"active"` | Semantic status |

Returns SVG string. On provider error, returns a badge with value `"error"`.

### `hw_kit`

Compose a full artifact kit. Returns dict of SVG strings keyed by artifact name.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `type` | `str` | `"readme"` | Kit type |
| `genome` | `str` | `"brutalist"` | Genome slug |
| `project` | `str` | `""` | Project name |
| `badges` | `str` | `""` | Badge pairs: `"build:passing,version:v0.6.3"` |
| `social` | `str` | `""` | Social glyph IDs: `"github,discord,x"` |

Example:

```
hw_kit(
  type="readme",
  genome="brutalist",
  project="my-project",
  badges="build:passing,version:v0.6.3,coverage:95%",
  social="github,discord,x"
)
```

This returns keys such as `badge-build`, `badge-version`, `badge-coverage`,
`strip`, `divider`, `icon-github`, `icon-discord`, and `icon-x`.

### `hw_discover`

Discover available HyperWeave components. Returns structured data about genomes, motions, glyphs, and frame types.

| Parameter | Type | Default | Description |
|---|---|---|---|
| `what` | `str` | `"all"` | `all`, `genomes`, `motions`, `glyphs`, `frames`, `matrix`, `url_grammar` |

Use `hw_discover` when an agent needs current registry data before composing:

```
hw_discover(what="genomes")
hw_discover(what="matrix")
hw_discover(what="url_grammar")
```

`what="genomes"` returns every loaded genome, including telemetry skins such as
`telemetry-codex`, `telemetry-claude-code`, and `telemetry-antigravity`.
`what="matrix"` returns matrix cell kinds, inferred-kind notes, available
presets, and projection support.

## Resources

### `hyperweave://schema`

ComposeSpec parameter reference. Returns JSON with all valid parameter values:

```json
{
  "type": ["badge", "strip", "icon", "divider", "marquee-horizontal", "receipt", "rhythm-strip", "stats", "chart", "matrix"],
  "genome": ["brutalist", "chrome", "automata", "primer"],
  "motion": ["static", "chromatic-pulse", ...],
  "state": ["active", "passing", ...],
  "glyph_mode": ["auto", "fill", "wire", "none"],
  "regime": ["normal", "permissive", "ungoverned"],
  "divider_variant": ["block", "current", "takeoff", "void", "zeropoint", "dissolve", "band", "seam", "sigil", "aura"]
}
```

The schema resource is the compact boundary reference for typed enum fields. Use
`hyperweave://genomes` or `hw_discover(what="genomes")` when you need the full
loaded genome registry, including telemetry-only skins.

### `hyperweave://genomes`

Full genome configurations with colors, motions, and profiles. Returns JSON keyed by genome ID.

### `hyperweave://motions`

Motion primitives with frame compatibility and CIM compliance. Returns JSON keyed by motion ID.
