## The Problem

You want artifacts that show real-time data -- GitHub stars, PyPI versions, npm
downloads, supply-chain scores -- rendered in your HyperWeave genome instead of
a third-party badge service.

## Data Token Grammar

HyperWeave uses one comma-separated data-token grammar across the CLI, HTTP API,
and MCP:

| Form | Meaning |
|---|---|
| `text:STRING` | Raw display text |
| `kv:KEY=VALUE` | Static labeled value |
| `<provider>:<identifier>.<metric>` | Live provider metric |

Provider aliases include `gh` / `github`, `pypi`, `npm`, `hf` /
`huggingface`, `arxiv`, `docker`, `crates` / `cargo`, `scorecard`, and `dora`.
Embedded commas in `text:` and `kv:` payloads escape as `\,`.

## Data-Driven Badges

### CLI

```bash
# Static labeled value
hyperweave compose badge "VERSION" --data kv:VERSION=0.6.3 -g brutalist -o version.svg

# GitHub stars
hyperweave compose badge "STARS" --data gh:openai/codex.stars -g brutalist --glyph github -o stars.svg

# npm downloads
hyperweave compose badge "NPM" --data npm:express.downloads -g chrome -o npm-downloads.svg

# crates.io version
hyperweave compose badge "CRATE" --data crates:serde.version -g primer -o crate-version.svg
```

`hyperweave live` remains as a single-provider badge shortcut:

```bash
hyperweave live github openai/codex stars -g brutalist -o stars.svg
```

### HTTP API

Use the two-segment badge route with `?data=`:

```
GET /v1/badge/{title}/{genome}.{motion}?data=<provider>:<identifier>.<metric>
```

Examples:

```bash
curl "http://localhost:8000/v1/badge/STARS/brutalist.static?data=gh:openai/codex.stars&glyph=github"
curl "http://localhost:8000/v1/badge/VERSION/chrome.static?data=pypi:fastmcp.version"
curl "http://localhost:8000/v1/badge/TRUST/primer.static?data=scorecard:openai/codex.score"
```

### MCP

```
hw_compose(type="badge", title="STARS", data="gh:openai/codex.stars", genome="brutalist", glyph="github")
```

For MCP clients that only need a single provider-backed badge, `hw_live` is also
available:

```
hw_live(provider="github", identifier="openai/codex", metric="stars", genome="brutalist")
```

## Supported Providers

| Provider | Token prefix | Identifier format | Example metrics |
|---|---|---|---|
| GitHub | `gh:` or `github:` | `owner/repo` | `stars`, `forks`, `watchers`, `issues`, `license`, `language`, `last_push`, `build`, `contributors`, `pull_requests` |
| PyPI | `pypi:` | `package-name` | `version`, `downloads`, `license`, `python_requires` |
| npm | `npm:` | `package-name` | `version`, `downloads`, `license` |
| arXiv | `arxiv:` | `paper-id` | `title`, `authors`, `categories`, `summary`, `published`, `updated`, `journal_ref`, `doi` |
| HuggingFace | `hf:` or `huggingface:` | `org/model` | `downloads`, `likes`, `tags`, `library_name`, `pipeline_tag`, `last_modified`, `gated`, `license` |
| Docker Hub | `docker:` | `namespace/repo` | `pull_count`, `star_count`, `last_updated` |
| crates.io | `crates:` or `cargo:` | `crate-name` | `version`, `downloads`, `recent_downloads`, `license` |
| OpenSSF Scorecard | `scorecard:` | `owner/repo` | `score`, `maintained`, `code_review`, `branch_protection`, `vulnerabilities`, `ci_tests` |
| DORA | `dora:` | `owner/repo` | `deploy_frequency`, `lead_time`, `change_failure_rate`, `mttr` |

## Data-Driven Strips

Strips support static metrics, live tokens, or a mix of both. Multiple live
provider tokens are fetched concurrently.

```
GET /v1/strip/{title}/{genome}.{motion}?data=provider:id.metric,provider:id.metric
```

```bash
# Static values
curl "http://localhost:8000/v1/strip/project/brutalist.static?data=kv:STARS=42K,kv:VERSION=0.6.3"

# Multiple providers
curl "http://localhost:8000/v1/strip/codex/brutalist.static?data=gh:openai/codex.stars,pypi:fastmcp.version"
```

In CLI form:

```bash
hyperweave compose strip "codex" --data gh:openai/codex.stars,pypi:fastmcp.version -g brutalist -o strip.svg
```

The resolved values are formatted as metric pairs and passed to the strip
resolver. If both `value` and `data` are provided, `data` wins when it resolves
to displayable values.

## Data-Driven Marquees

Marquees can mix text tokens and live values:

```
GET /v1/marquee/{title}/{genome}.{motion}?data=text:NEW,gh:owner/repo.stars
```

```bash
curl "http://localhost:8000/v1/marquee/live/primer.static?data=text:NEW,gh:openai/codex.stars,text:DOWNLOADS"

hyperweave compose marquee-horizontal --data text:NEW,gh:openai/codex.stars,text:DOWNLOADS -g primer -o marquee.svg
```

## Error Handling

Malformed data-token strings render the SMPTE error SVG with HTTP 200 and
`X-HW-Error-Code: 400` so README image proxies still display a useful artifact.

If a single live provider fetch fails, that token degrades to `"--"` while
successful tokens render normally. The failure cache TTL is short so recovered
providers can refresh quickly.

## Caching

Provider TTLs vary by connector and are carried through the response cache. For
multi-provider artifacts, the HTTP `Cache-Control` TTL is the minimum across all
resolved live tokens and includes `stale-while-revalidate=3600`.

Common TTLs:

| Provider family | Typical TTL |
|---|---|
| GitHub scalar metrics | 300s |
| GitHub build status | 120s |
| PyPI, npm, HuggingFace, Docker Hub, crates.io | 600s |
| arXiv | 1800s |
| OpenSSF Scorecard | 21600s |
| DORA | 600s |
| Failed live token | 60s |
