## The Problem

You want visible motion on badges, strips, or icons, but need to know which
motions work with each frame, which genomes support them, and how CIM policy
changes the rendered output.

HyperWeave's current public motion vocabulary is `static` plus five SVG-native
SMIL border motions.

## Border Motions

Border motions attach an SVG overlay to the artifact frame when both frame and
genome compatibility allow it.

| Motion | Applies To | Effect |
|---|---|---|
| `chromatic-pulse` | `badge`, `strip`, `icon` | Chromatic gradient cycling with stroke-width and opacity modulation |
| `corner-trace` | `badge`, `strip`, `icon` | Single runner tracing the rounded frame perimeter |
| `dual-orbit` | `badge`, `strip`, `icon` | Two perimeter runners offset by half a circuit |
| `entanglement` | `badge`, `strip`, `icon` | Coupled chromatic gradient and paired dash traversal |
| `rimrun` | `badge` | Badge-sized frosted-prism rim runners with refraction and glare layers |

```bash
hyperweave compose badge "build" "passing" -g brutalist -m chromatic-pulse --regime permissive
hyperweave compose strip "readme-ai" "" -g chrome --metrics "STARS:2.9k,FORKS:278" -m dual-orbit --regime permissive
hyperweave compose icon "" "" --glyph github -g brutalist -m corner-trace --regime permissive
```

## Frame Compatibility

Frame compatibility is checked separately from CIM policy. `static` works on
every frame.

| Frame | Border Motions |
|---|---|
| `badge` | `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement`, `rimrun` |
| `strip` | `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement` |
| `icon` | `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement` |
| Other frames | `static` only |

## CIM Compliance

The Compositional Integrity Model (CIM) defines which motions are safe for
governed contexts.

**CIM-compliant**: `static`

**Non-CIM**: `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement`,
`rimrun`

```bash
# normal accepts the request but renders static output
hyperweave compose badge "build" "passing" -g brutalist -m dual-orbit

# permissive renders the requested visible border motion
hyperweave compose badge "build" "passing" -g brutalist -m dual-orbit --regime permissive
```

Under `normal`, non-CIM motions are corrected to `static`. Under `permissive`,
the requested border motion renders and HyperWeave records the policy violation.
Under `ungoverned`, policy-lane checks are skipped, but visible output still
requires a registered motion config and template.

## HTTP API

Motion is specified in the URL grammar using dot notation:

```
/v1/badge/{title}/{value}/{genome}.{motion}?regime=permissive
/v1/strip/{title}/{genome}.{motion}?value=...&regime=permissive
/v1/icon/{glyph}/{genome}.{motion}?regime=permissive
```

```bash
curl "localhost:8000/v1/badge/build/passing/brutalist.chromatic-pulse?regime=permissive"
curl "localhost:8000/v1/strip/readme-ai/chrome.dual-orbit?value=STARS:2.9k,FORKS:278&regime=permissive"
curl "localhost:8000/v1/icon/github/brutalist.corner-trace?regime=permissive"
```

## Genome Compatibility

Each genome declares its own compatible motion list. A motion must be supported
by both the selected genome and the selected frame.

| Genome | Compatible Motions |
|---|---|
| `brutalist` | `static`, `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement`, `rimrun` |
| `chrome` | `static`, `chromatic-pulse`, `corner-trace`, `dual-orbit`, `entanglement`, `rimrun` |
| `automata` | `static`, `chromatic-pulse`, `corner-trace`, `rimrun` |
| `primer` | `static`, `chromatic-pulse`, `corner-trace` |
| `telemetry-*` | `static` |

For example, `rimrun` is available in the `brutalist`, `chrome`, and `automata`
genome lists, but it is still badge-scoped by frame compatibility.
